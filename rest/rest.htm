What Is REST

REST stands for Representational State Transfer.

REST is an software architectural style for designing application not a protocol, and typically it uses simple HTTP (but not always ) for data communication which is stateless and client-server protocol .

REST was initially proposed by Roy Thomas Fielding in 2000


REST Architecture

In Rest architectural every component is a resource and a resource is accessed by a common interface using HTTP standard methods (GET, POST, PUT, DELETE, etc.). Each resource is identified by URIs (Uniform resource Identifier)/ global IDs. Any client can ask for any of resource using resource URI.


RESTFul WEB Service

RESTful Web Services are REST architecture based web services.

RESTful web services are lightweight, highly scalable and maintainable and are very commonly used to create APIs for web based applications. Despite being simple, REST is fully-featured and alternative to mechanisms like RPC (Remote Procedure Calls) and Web Services (SOAP, WSDL, etc.).

Web service developed as rest API can be accessed over http interface by any client developed in any language. Rest API can provide result in different formate in Plain Text, HTML, XML and JSON.

RESTful applications typically use HTTP requests to post data (create and/or update - POST/PUT Method), read data (e.g., make queries -GET Method), and delete (DELETE Method)data. Thus, REST uses HTTP for all four CRUD (Create/Read/Update/Delete) operations

REST service is

  1. Platform-independent (Server might be Unix/Windows, the client might be a Mac, or anything else)
  2. Language-independent (java can talk to c++, etc.)
  3. Standards-based (runs on top of HTTP)


HTTP Methods

Following well known HTTP methods are commonly used in REST based architecture.

  • GET - Provides a read only access to a resource.
  • POST - ----------
  • PUT - Used to create a new resource.
  • DELETE - Used to remove a resource.
  • OPTIONS - Used to get the supported operations on a resource.
