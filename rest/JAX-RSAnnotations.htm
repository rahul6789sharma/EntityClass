JAX-RS - Annotations

JAX-RS (JSR 311) is the specification to develop RESTful web service. JAX-RS stands for JAVA API for RESTful Web Services.Its 2.0 version was released in 24 May 2013. In this tutorial we will use Jersey as implementation of JAX-RS specification.

Below are the most commonly used annotations that are used to map a resource as a web service resource.

REST follows one-to-one mapping between create, read, update, and delete (CRUD) operations and HTTP methods.

  • To create a resource on the server, use POST.
  • To retrieve a resource, use GET.
  • To change the state of a resource or to update it, use PUT.
  • To remove or delete a resource, use DELETE.

Annotation Description
@path Define relative URI path of the resource. It can be specified on class or method
@GET Define method that responds to GET HTTP request. commonly used to fetch resource.
@PUT Define method that responds to PUT HTTP request. commonly used to create resource.
@POST Define method that responds to POST HTTP request. commonly used to create/update resource.
@DELETE Define method that responds to DELETE HTTP request. commonly used to delete resource.
@HEAD Define method that responds to HEAD HTTP request. commonly used to get status of method availability.
@Produces Specify the MIME media types of a resource that can be produce and send back to the client by web service. such as APPLICATION/XML, TEXT/HTML, APPLICATION/JSON etc.
@Consumes Specify the MIME media types that can be consumed/accepted by a resource during POST request. such as APPLICATION/XML, TEXT/HTML, APPLICATION/JSON etc.
@PathParam Bind REST-style URI parameters to method arguments
@QueryParam Request parameters in query string of URI and passed to a method arguments.
@FormParam Used to read parameters sent in POST requests directly
